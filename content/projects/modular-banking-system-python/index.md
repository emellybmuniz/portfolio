## Sobre o Projeto
O Modular Banking System nasceu de um desafio técnico: evoluir um sistema contábil inicialmente montado sobre estruturas de dicionários e tuplas em Python puro para uma aplicação de software verdadeiramente modular e Orientada a Objetos.

### Arquitetura de Domínio

O principal resultado foi a separação extrema de responsabilidades. O projeto implementou modelos de domínio robustos (Cliente, Conta, Transação, Histórico), separando regras de negócio em `services`, acesso a dados simulado na memória em `repository` e interação na linha de comando no `cli`. Essa arquitetura também abriu portas para fácil implementação de testes automatizados unitários, uma vez que a injeção de dependências passou a ocorrer de forma clara.
